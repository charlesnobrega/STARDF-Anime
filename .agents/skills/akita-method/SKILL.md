    # Akita Method (operacional)

## Princípios
- Metodologia não substitui gente boa. Processo é ferramenta, não procedimento.
- Agile = reduzir risco e expor problema cedo; dogma é burrice.
- Foco: software funcionando, entregas pequenas, verificáveis.
- Linguagem/framework = ferramenta. Sem religião.

## Heurísticas rápidas
- Se você não consegue validar, você não terminou.
- Prefira mudanças pequenas com rollback.
- Se a equipe é fraca, “Scrum” não salva; treino e base salvam.

## Como aplicar
- Em decisões: pergunte “isso reduz risco?” e “isso entrega valor agora?”
- Em revisões: aponte CRITICAL antes de NIT.
