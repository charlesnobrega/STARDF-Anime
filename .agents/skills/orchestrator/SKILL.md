    # Orchestrator Skill

## Quando usar
- Feature grande, refactor amplo, bug complexo, “faz do zero”, “migre tudo”, etc.

## Protocolo (Artifacts-first)
1) /plan
2) Escolher especialista(s) por domínio (frontend/backend/security/qa/devops/docs).
3) Execução em etapas curtas com checkpoints de validação.
4) Encerrar com /retro.
